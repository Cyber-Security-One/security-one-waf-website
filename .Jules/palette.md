## 2026-03-05 - Accessible Custom Cards
**Learning:** Custom interactive `div` elements (acting as cards or buttons) are completely invisible to keyboard users and lack semantics for screen readers unless explicitly added. The existing `FeatureCard` component relied solely on `@click` and mouse events (`@mouseenter`, `@mouseleave`), ignoring keyboard navigation and focus states.
**Action:** When creating custom interactive elements, always ensure they have `role="button"`, `tabindex="0"`, keyboard event handlers (`@keydown.enter`, `@keydown.space`), and appropriate `aria-label` attributes. Furthermore, map focus events (`@focus`, `@blur`) to the existing hover state logic to ensure keyboard users receive the same visual feedback as mouse users, alongside a distinct `:focus-visible` outline.
