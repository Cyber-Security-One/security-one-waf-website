## 1. 系統概述

Security One SOC 是一個功能完整的企業級 Web 應用程式防火牆，主要特點：

- **多站點支援**：可同時保護多個網站與網域
- **雙重運作模式**：防禦模式（即時封鎖）或觀察模式（僅記錄）
- **多層防禦架構**：結合規則引擎、AI 分析與行為偵測
- **視覺化管理介面**：現代化的 Vue.js 單頁應用程式
- **容器化部署**：Docker 容器化，易於安裝與擴展
- **IDS/IPS 整合**：分散式入侵偵測與防禦系統

## 2. 儀表板與監控

### 功能說明
儀表板提供即時的安全態勢概覽，讓管理員快速掌握系統狀態。

### 詳細功能
| 功能項目 | 說明 |
|---------|------|

### 功能說明

### 詳細功能

#### 3.1 站點基本設定
| 設定項目 | 說明 | 選項 |
|---------|------|------|
| **站點名稱** | 易於辨識的站點名稱 | 自訂文字 |
| **網域名稱** | 站點的完整網域 | 例：`example.com` |

| 模式 | 說明 |
|-----|------|
| **防禦模式 (Defense)** | 即時攔截攻擊請求並記錄日誌 |
| **觀察模式 (Observation)** | 僅記錄攻擊日誌但不阻擋，適合初期部署測試 |

#### 3.3 防護等級
| 等級 | 說明 |
|-----|------|
| **Off** | 關閉所有 WAF 防護 |
| **Low** | 僅攔截明確的攻擊行為 |
| **Medium** | 平衡防護與誤報率（建議值） |
| **High** | 最嚴格的防護等級，可能增加誤報 |

#### 3.4 進階功能
- **強制 HTTPS**：自動將 HTTP 重定向至 HTTPS
- **後端 SSL**：使用 SSL 連接後端伺服器
- **CSP 安全標頭**：啟用 Content Security Policy
- **組態重生成**：手動重新產生 Nginx 配置檔

## 4. 負載均衡

### 功能說明
支援多後端伺服器的負載均衡功能，確保高可用性。

### 負載均衡策略
| 策略 | 說明 |
|-----|------|
| **Round Robin** | 輪詢分配，依序將請求分配給各伺服器 |
| **Random** | 隨機分配到任一可用伺服器 |
| **Least Connections** | 優先分配給連線數最少的伺服器 |
| **IP Hash** | 基於來源 IP 分配，確保同一用戶訪問同一伺服器（會話持久性） |

### Upstream 管理
| 功能 | 說明 |
|-----|------|
| **新增後端** | 新增多個後端伺服器 |
| **權重設定** | 設定各伺服器的流量權重 (Weight) |
| **健康檢查** | 自動偵測並移除異常伺服器 |
| **動態調整** | 不停機情況下新增或移除後端 |

## 6. 核心攻擊防護

### 功能說明
規則引擎提供的傳統攻擊特徵碼偵測與防護。

### 防護類型

#### 6.1 SQL Injection (SQL 注入)
- **偵測目標**：URL、Query String、POST Body、Headers
- **防護內容**：
  - 常見 SQL 關鍵字（SELECT, UNION, INSERT, DELETE 等）
  - SQL 註解符號（--, /\*\*/, #）
  - 資料庫函數（CONCAT, SUBSTR, SLEEP 等）
  - 編碼繞過嘗試（Hex, URL Encoding）

#### 6.2 Cross-Site Scripting (XSS)
- **偵測目標**：URL、Query String、POST Body
- **防護內容**：
  - `<script>` 標籤
  - 事件處理器（onclick, onerror, onload 等）
  - JavaScript 協議（javascript:）
  - 編碼繞過嘗試

#### 6.3 Path Traversal (路徑遍歷)
- **偵測目標**：URL 路徑
- **防護內容**：
  - 目錄跳轉序列（../, ..\）
  - 編碼變體（%2e%2e/, %252e 等）
  - 敏感檔案存取（/etc/passwd, web.config 等）

#### 6.4 Remote Code Execution (RCE)
- **偵測目標**：URL、Query String、POST Body
- **防護內容**：
  - Shell 命令注入（; | && 等）
  - 常見命令（wget, curl, nc, bash 等）
  - 反向 Shell 嘗試

#### 6.5 Scanner Detection (掃描器偵測)
- **偵測目標**：User-Agent、請求模式
- **防護內容**：
  - 已知掃描器特徵（nikto, sqlmap, nmap 等）
  - 自動化工具識別
  - 異常請求頻率

#### 6.6 OGNL Injection (Java 專用)
- **偵測目標**：URL、Query String、POST Body
- **防護內容**：
  - OGNL 表達式模式
  - Struts2 相關漏洞特徵
  - Java 物件存取嘗試

#### 6.7 Java Deserialization (Java 反序列化)
- **偵測目標**：Request Body、Headers
- **防護內容**：
  - 序列化魔術位元組
  - 常見 Gadget Chain 特徵
  - Base64 編碼的序列化資料

### 防護設定控制
每種攻擊類型都可獨立開啟或關閉，提供靈活的防護策略。

## 13. 站點級身份認證

### 功能說明
無需修改後端程式碼，即可在 WAF 層為特定路徑加上登入驗證保護。

### 功能特點

#### 13.1 路徑保護
| 功能 | 說明 |
|-----|------|
| **路徑設定** | 指定需要保護的 URL 路徑（支援萬用字元） |
| **方法設定** | 可指定保護的 HTTP 方法（GET/POST/ALL） |
| **多路徑保護** | 可同時保護多個路徑 |

#### 13.2 使用者管理
| 功能 | 說明 |
|-----|------|
| **帳號建立** | 為站點建立專屬使用者帳號 |
| **密碼管理** | 使用 bcrypt 加密儲存密碼 |
| **多帳號支援** | 每個站點可建立多個使用者 |

#### 13.3 Passkey 無密碼登入
| 功能 | 說明 |
|-----|------|
| **WebAuthn 支援** | 支援 FIDO2/WebAuthn 標準 |
| **生物辨識** | 支援指紋、Face ID 等生物辨識 |
| **硬體金鑰** | 支援 YubiKey 等硬體安全金鑰 |
| **多裝置** | 使用者可註冊多個 Passkey |
| **Passkey 管理** | 使用者可自行管理已註冊的 Passkey |

### 認證流程
1. 用戶訪問受保護路徑

## 34. IDS Agent 偵測引擎

### 功能說明
IDS Agent 內建三種偵測引擎，提供多層次的威脅偵測能力。

### 34.1 特徵偵測引擎 (Signature Engine)

#### 功能說明
基於已知攻擊特徵碼的規則匹配偵測。

#### 偵測能力
| 攻擊類型 | 偵測內容 |
|---------|---------|
| **SQL 注入** | SQL 關鍵字、函數、語法結構 |
| **XSS** | Script 標籤、事件處理器、協議繞過 |
| **路徑遍歷** | 目錄跳轉序列、敏感檔案存取 |
| **命令注入** | Shell 命令、管道符號、反引號 |
| **PHP 注入** | PHP 函數、包含語句 |
| **WebShell** | 後門特徵、執行函數 |
| **SSRF** | 內網 IP、雲元數據 URL |
| **XXE** | XML 實體宣告、外部實體 |
| **Log4Shell** | JNDI 注入模式 |

#### 技術特點
- 正規表達式匹配
- 資料庫與內建規則並用
- 規則快取優化
- 批量分析支援

### 34.2 異常偵測引擎 (Anomaly Engine)

#### 功能說明
基於基線偏離的異常行為偵測。

#### 偵測項目
| 異常類型 | 說明 | 閾值 |
|---------|------|-----|
| **高請求頻率** | 單一 IP 請求過快 | 100 req/min |
| **大請求體** | 請求大小異常 | 1MB |
| **長 URL** | URL 長度超標 | 2048 字元 |
| **可疑 UA** | 空白或異常 User-Agent | - |
| **非營業時段** | 凌晨高頻請求 | 8AM-10PM 外 |

#### 嚴重度計算
根據異常程度動態計算：
- **Critical**：超過閾值 2 倍以上
- **High**：超過閾值 1.5 倍
- **Medium**：超過閾值
- **Low**：接近閾值

### 34.3 行為分析引擎 (Behavior Engine)

#### 功能說明
分析請求行為模式，偵測惡意行為。

#### 偵測項目
| 行為類型 | 說明 | 觸發條件 |
|---------|------|---------|
| **掃描行為** | 快速存取大量 URL | >20 URL/5分鐘 |
| **暴力破解** | 登入失敗次數過多 | >5 次失敗 |
| **爬蟲行為** | 自動化工具識別 | Headless/Selenium |
| **快速請求** | 高頻連續請求 | >10 req/10秒 |

#### 行為評分
- 0-100 分制
- 暴力破解最高加 50 分
- 掃描行為最高加 40 分
- 爬蟲行為加 20 分
- 快速請求最高加 30 分

## 46. IPS 入侵防禦引擎（Snort + Suricata）

### 功能說明

|-----|------|
| **Snort** | 經典的開源 IDS/IPS 引擎，基於規則的網路流量分析 |

| **運行統計** | Suricata 效能統計數據 |

### 運作模式
| 模式 | 說明 |
|-----|------|
| **IDS 模式** | 僅偵測與告警，不阻擋流量 |

### IPS 控制操作
| 操作 | 說明 |

| **全部停用** | 一鍵停用所有 Agent 的 IPS |
| **全域預設模式** | 設定預設 IPS 模式，同步到所有 Agent |

### Snort 規則管理
| 功能 | 說明 |
|-----|------|
| **規則新增** | 建立新的偵測規則（SID、動作、協議、來源/目的） |

### Snort 規則欄位
