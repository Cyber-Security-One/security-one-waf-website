## 1. 系統概述

Security One WAF 是一個功能完整的企業級 Web 應用程式防火牆，主要特點：

- **多站點支援**：可同時保護多個網站與網域
- **雙重運作模式**：防禦模式（即時封鎖）或觀察模式（僅記錄）
- **多層防禦架構**：結合規則引擎、AI 分析與行為偵測
- **視覺化管理介面**：現代化的 Vue.js 單頁應用程式
- **容器化部署**：Docker 容器化，易於安裝與擴展

## 2. 儀表板與監控

### 功能說明

## 4. 負載均衡

### 功能說明
支援多後端伺服器的負載均衡功能，確保高可用性。

### 負載均衡策略
| 策略 | 說明 |
|-----|------|
| **Round Robin** | 輪詢分配，依序將請求分配給各伺服器 |
| **Random** | 隨機分配到任一可用伺服器 |
| **Least Connections** | 優先分配給連線數最少的伺服器 |
| **IP Hash** | 基於來源 IP 分配，確保同一用戶訪問同一伺服器（會話持久性） |

### Upstream 管理
| 功能 | 說明 |
|-----|------|
| **新增後端** | 新增多個後端伺服器 |
| **權重設定** | 設定各伺服器的流量權重 (Weight) |
| **健康檢查** | 自動偵測並移除異常伺服器 |
| **動態調整** | 不停機情況下新增或移除後端 |

## 6. 核心攻擊防護

### 功能說明
規則引擎提供的傳統攻擊特徵碼偵測與防護。

### 防護類型

#### 6.1 SQL Injection (SQL 注入)
- **偵測目標**：URL、Query String、POST Body、Headers
- **防護內容**：
  - 常見 SQL 關鍵字（SELECT, UNION, INSERT, DELETE 等）
  - SQL 註解符號（--, /\*\*/, #）
  - 資料庫函數（CONCAT, SUBSTR, SLEEP 等）
  - 編碼繞過嘗試（Hex, URL Encoding）

#### 6.2 Cross-Site Scripting (XSS)
- **偵測目標**：URL、Query String、POST Body
- **防護內容**：
  - `<script>` 標籤
  - 事件處理器（onclick, onerror, onload 等）
  - JavaScript 協議（javascript:）
  - 編碼繞過嘗試

#### 6.3 Path Traversal (路徑遍歷)
- **偵測目標**：URL 路徑
- **防護內容**：
  - 目錄跳轉序列（../, ..\）
  - 編碼變體（%2e%2e/, %252e 等）
  - 敏感檔案存取（/etc/passwd, web.config 等）

#### 6.4 Remote Code Execution (RCE)
- **偵測目標**：URL、Query String、POST Body
- **防護內容**：
  - Shell 命令注入（; | && 等）
  - 常見命令（wget, curl, nc, bash 等）
  - 反向 Shell 嘗試

#### 6.5 Scanner Detection (掃描器偵測)
- **偵測目標**：User-Agent、請求模式
- **防護內容**：
  - 已知掃描器特徵（nikto, sqlmap, nmap 等）
  - 自動化工具識別
  - 異常請求頻率

#### 6.6 OGNL Injection (Java 專用)
- **偵測目標**：URL、Query String、POST Body
- **防護內容**：
  - OGNL 表達式模式
  - Struts2 相關漏洞特徵
  - Java 物件存取嘗試

#### 6.7 Java Deserialization (Java 反序列化)
- **偵測目標**：Request Body、Headers
- **防護內容**：
  - 序列化魔術位元組
  - 常見 Gadget Chain 特徵
  - Base64 編碼的序列化資料

### 防護設定控制
每種攻擊類型都可獨立開啟或關閉，提供靈活的防護策略。

## 8. CC/DDoS 防護

### 功能說明
針對 Challenge Collapsar 與分散式拒絕服務攻擊的防護機制。

### 詳細設定
| 設定項目 | 說明 | 預設值 |
|---------|------|-------|
| **啟用狀態** | 開啟或關閉 CC 防護 | 啟用 |
| **速率限制** | 單一 IP 每秒最大請求數 | 100 |

| 動作 | 說明 |
|-----|------|
| **Block** | 直接拒絕請求 |
| **Captcha** | 顯示人機驗證頁面 |
| **Delay** | 延遲回應以降低攻擊效率 |
| **Log** | 僅記錄但不採取動作 |

### 統計資訊
- 今日觸發次數
- 當前封鎖 IP 數
- CAPTCHA 通過數量

## 11. 自動封鎖規則

### 功能說明

### 規則設定
| 設定項目 | 說明 | 範圍 |

| **封鎖時間** | 封鎖的持續時間 | 60-604800 秒 |
| **優先級** | 規則的執行優先順序 | 0-100 |
| **啟用狀態** | 規則是否生效 | 啟用/停用 |

### 支援的攻擊類型
- 所有攻擊
- SQL 注入
- XSS
- 路徑遍歷
- CC 攻擊
- 掃描器
- RCE
- OGNL 注入
- Java 反序列化
- 自訂規則

### 使用範例

## 13. 站點級身份認證

### 功能說明
無需修改後端程式碼，即可在 WAF 層為特定路徑加上登入驗證保護。

### 功能特點

#### 13.1 路徑保護
| 功能 | 說明 |
|-----|------|
| **路徑設定** | 指定需要保護的 URL 路徑（支援萬用字元） |
| **方法設定** | 可指定保護的 HTTP 方法（GET/POST/ALL） |
| **多路徑保護** | 可同時保護多個路徑 |

#### 13.2 使用者管理
| 功能 | 說明 |
|-----|------|
| **帳號建立** | 為站點建立專屬使用者帳號 |
| **密碼管理** | 使用 bcrypt 加密儲存密碼 |
| **多帳號支援** | 每個站點可建立多個使用者 |

#### 13.3 Passkey 無密碼登入
| 功能 | 說明 |
|-----|------|
| **WebAuthn 支援** | 支援 FIDO2/WebAuthn 標準 |
| **生物辨識** | 支援指紋、Face ID 等生物辨識 |
| **硬體金鑰** | 支援 YubiKey 等硬體安全金鑰 |
| **多裝置** | 使用者可註冊多個 Passkey |
| **Passkey 管理** | 使用者可自行管理已註冊的 Passkey |

### 認證流程
1. 用戶訪問受保護路徑
2. WAF 重導向至登入頁面
3. 用戶選擇密碼登入或 Passkey 登入
4. 驗證成功後發放 Token Cookie
5. Cookie 有效期 24 小時

## 15. 攻擊日誌與分析

### 功能說明
完整記錄所有偵測到的攻擊事件，並提供豐富的分析功能。

### 日誌欄位
| 欄位 | 說明 |
|-----|------|

| **IP 搜尋** | 搜尋特定 IP 的攻擊記錄 |

### 統計資訊
- 總攻擊數
- 今日攻擊數
- 今日封鎖數
- 今日高危攻擊數

## 22. AI 規則建議

### 功能說明

### 功能詳情

## 26. 技術規格

### 系統架構
| 項目 | 技術 |
|-----|------|
| **部署方式** | Docker 容器化 |
| **反向代理** | OpenResty (Nginx + Lua) |
| **應用框架** | Laravel 12.x (PHP 8.3+) |
| **前端框架** | Vue.js 3 + Inertia.js v2 |
| **CSS 框架** | Tailwind CSS v4 |
| **資料庫** | SQLite（預設）/ MySQL / PostgreSQL |
| **AI 引擎** | Ollama (本地 LLM) |
| **即時通訊** | Server-Sent Events (SSE) |
| **3D 視覺化** | Three.js |

### 容器組成
| 容器 | 用途 |
|-----|------|
| **security-one-waf** | 主應用程式與管理介面 |
| **security-one-waf-proxy** | OpenResty 反向代理與 WAF 引擎 |

### 網路架構
| 網路 | 用途 |
|-----|------|
| **security-one-network** | WAF 內部組件通訊 |
| **protected-network** | 連接受保護的後端站點 |

### 效能特性
- 高效能 OpenResty 核心，支援高併發
- 規則引擎使用 Lua 腳本優化
- AI 分析可非同步處理避免阻塞
- 前端 SPA 架構提升使用體驗
- SSE 即時資料推送

### 安全特性
- 密碼使用 bcrypt 加密儲存
- CSRF Token 保護
- XSS 防護標頭
- HTTP-Only Cookie
- WebAuthn/Passkey 雙因素認證
- Let's Encrypt SSL 自動簽發
